# Scene 4.5 — 十四天 / Fourteen days

## 戏剧问题 / Dramatic Question
*Will the steady state — the one Chen Mo fought to return to — feel like anything once he is in it?*
他奋力夺回的"正常"，回到之后会是什么感觉？

## 目标 / Objective
**回到岗位** —— 完成账号恢复手续；接受派单；把生活复位。
（这是一个**没有阻力**的目标——这正是讽刺。1.1 序列开场他也在送外卖，那时有差评、有夜雨；本场没有任何坏事——也没有任何好事。）

## 冲突 / Conflict
- **外部**：几乎没有。平台账号恢复评级用了 11 天到 4.7（接近他冻结前的 4.8）；调度系统语音和过去一样平直；天气、订单、楼栋、客户都不再叫他的名字也不再不叫他的名字——**他已经回到隐形里**。
- **内部**：他**期待**会有某种调整期——某种"重新适应"的不适感。没有。他第二天就送了第一单（步骤性的，蒙太奇一闪），第七天就回到日常节奏。这个"无缝接回"是本场真正的冲突——**他想找到至少一个不适的征兆**，找不到。
- **层级**：[[inner-conflict]] 主导，但表现为**外部的过于顺滑**。

## 转折点 / Turning Point
> 第 23 天 17:42（与激励事件第 3 天 18:47 的时间形成一小时差对位）。App 弹窗。他看见地址："建国南路 47 弄 6 号楼 603 室"——**和激励事件的 602 室同一栋楼，隔壁一户，**他过去无数次擦肩而过这道门。他**没有犹豫**地按下"接单"。这一"没有犹豫"是本场不可逆拍：故事此刻确认，他不仅回到了岗位，**他真的什么都没带回来**——连一点应该有的犹豫都没有。

## 价值翻转 / Value Shift
- **命名价值**：+3 / −3 持平，**数值不动**——但状态从"刚发生的剪刀差"翻转为"稳定的常态剪刀差"。
- 这是 [[resolution]] 必需的"沉淀"——意义在 4.6 才会被图像兑现，但本场把读者推到那扇门前。
- 审计豁免理由：本场是 Resolution 的双场结构第一半。McKee Ch.13 允许 Resolution 的最后一拍是双场（沉淀 + 图像）。

## 种植项 / Planted Items
- **App 名字栏第 3 次出现**（按 misdirection-plan "第三次写最慢"）—— 建议放在本场中段：他在出租屋桌前，把账号评级看到回到 4.7，然后**他看着名字栏停了一会**。**陈默。** 字号、颜色、位置和 PC-1 冻结时的、4.3 恢复时的完全一致——也和 4.6 在 603 门外看到的将会一致。**名字从来不是问题**。
- **激励事件的物品镜像**：1.2 那两杯奶茶 + 炸鸡 → 本场弹窗"麻辣烫 / 柠檬茶两杯"——**柠檬茶两杯**是关键回响（奶茶两杯—柠檬茶两杯的对位）。同一栋楼，隔壁一户，平价小吃。脊椎 §9 第 5 条强制要求的镜像。
- **6 楼**——同一栋楼，**他必须再一次爬这道楼梯**（电梯依然故障——为 4.6 的物理回响建立）。

## 还款项 / Payoffs
- **PC-1 美团账号冻结画面** → 本场账号恢复至 4.7，**回到了开场之前的状态**。叙事弧外部完整闭环。
- **苏寒"你做的事，世界终于看见了。只是它叫你魔尊"** → 在本场以更狠的形式被验证："世界又不看见你了。它叫你陈默。"——这句话没有任何人说出，是 misdirection-plan 第二触发点的预备。
- **激励事件 18:47 / 6 楼 / 两单饮料 / 炸鸡** → 本场对位：17:42 / 6 楼（同一栋）/ 两杯柠檬茶 / 麻辣烫。同样的地理，几乎同样的时间。

## 图像/母题 / Image & Motif
- App 名字栏的**第 3 次出现**——这一次的处理：让他看着它**比前两次都久**。可考虑用一整段（150–250 字）只描述他坐着、屏幕亮着、那两个字在那里。**让读者也停下来看那两个字。**
- 14 天蒙太奇：极短（300–500 字）。建议拍法——前 3 天每天一句（一个小动作：他第一次再开 App 时的迟疑；他被一个常客点评"快"；他在拉面馆门口路过没有进去）；中段一句概括（"他没有去找过姐姐。姐姐也没找过他。"）；末了停在第 23 天的弹窗。
- **不出现的人物**：苏寒（电话号码他没打——那张外卖菜单他扔了还是没扔，不交代）；林晟（在他的世界外，可能已经回到学校）；姐姐（从未联系）；老太太（救活了，但他没去医院探望，因为**她也不认识他**）——这些"不出现"本身是 misdirection-plan 真相读法的承载。

## 裴鸣备注 / Persona Notes
- **决斗伪装成闲聊**：蒙太奇必须**伪装成日常**——没有戏剧性。裴鸣的"前面是绕路，然后一刀"在这里：所有的绕路（蒙太奇细节）都铺向最后那一下"接单"按钮。
- **真相库 #4（底层的诗意）**：这 14 天不是苦难。陈默在恢复的过程里**有他的尊严**——他按时上线、不投诉、不抱怨。这正是讽刺的精度——他的尊严就在这种自我维持里，**而这种尊严本身没有人在看**。裴鸣**不**让陈默感到他自己的尊严；让读者感到。
- **拒绝清单**：不写"他想起了那六分钟"；不写"他偶尔会拿出手机看苏寒留下的号码"（**他不会**）；不写"他经过 6 楼时心一紧"——蒙太奇期间他**多次**经过 6 楼，**毫无反应**——这才是讽刺。
- 句式：蒙太奇用名词短句堆叠（"早班。中班。雨。差评。好评。"）；接单瞬间用一个完整的简单句（"他按下'接单'。"）。

## 八点场景审计 / Eight-Point Scene Audit
- [x] 翻转：数值不动；状态翻转（剪刀差从"刚发生"变为"已稳定"）。这是 Resolution 双场结构的允许形式。
- [x] 冲突挣得：冲突在于"过于顺滑"——他无法找到一个不适。这种**找不到**就是冲突的胜方。
- [x] 转折点可定位：他按下"接单"的那一拍。
- [x] 场景目标可演：演员可以演"完成一个工作日，再完成下一个"——蒙太奇可演。
- [x] 鸿沟在场：他期望有调整期；没有。
- [x] 保守动作：他全程做最少的事。
- [x] 潜文本：表面是"日子继续"；潜文本是"日子继续是 Resolution 自己的判决"。
- [x] 服务脊椎：把读者推到 4.6 那扇门前；让 App 名字栏的第 3 次出现完成种植的二次触发。

**审计：通过。一点说明**：opening 与 closing 数值相同，理由如 4.1——属于 Resolution 双场结构的前半（沉淀场），最终翻转在 4.6 的图像兑现。如严格审计要求每场独立翻转，可考虑把"接单瞬间"独立成微型场景，但会失去蒙太奇的稳态感。**保留为单场。**
