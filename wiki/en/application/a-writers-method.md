# A Writer's Method (Inside-Out Workflow)

> 中文版：[[wiki/zh/application/a-writers-method|中文]]

## Overview
McKee's end-to-end project workflow. Six hypothetical months from idea to last polish. The method is designed to *postpone dialogue until events, values, and subtext are stress-tested*, so that dialogue, when it finally arrives, is character-specific and load-bearing.

```mermaid
flowchart LR
  idea[Idea]
  cards[[Step-Outline (4 months)]]
  pitch[Pitch Until Silence]
  tr[[Treatment (weeks)]]
  sp[Screenplay (5-10 pp/day)]
  pol[Polishes]
  idea --> cards
  cards --> pitch
  pitch -->|fail| cards
  pitch -->|silence| tr
  tr --> sp
  sp --> pol
  pol -->|reveal treatment faults| tr
```

## Steps
1. **Research and world-build** in a separate file cabinet. Biographies, history, vocabulary. This feeds the outline without bloating it.
2. **Step-Outline** on 3x5 cards, stacked per act. See [[step-outline]].
3. **Pitch** the ten-minute spoken version to intelligent, sensitive listeners. Iterate until the story, told in ten minutes, silences the room.
4. **Treatment.** Expand every card to a paragraph of present-tense action *plus* full subtext — **no dialogue**. See [[treatment]].
5. **Screenplay.** Convert treatment description to screen description; add dialogue. Dialogue at this stage is character-specific and delighted to be freed.
6. **Polishes.** If a line or scene doesn't work, do not rewrite dialogue — return to the treatment, fix setups, adjust payoffs, then re-emerge to the screenplay.

## Checklist
- [ ] Do not start the keyboard with scene description + dialogue.
- [ ] Step-outline on cards, stacks per act.
- [ ] Pitch-test until silence.
- [ ] Full-subtext treatment of 60–90 pages, zero dialogue.
- [ ] Screenplay runs 5–10 pages a day after treatment.
- [ ] Polishes go through the treatment, not the page.
