## 2. Canonical Chapter Rule / 规范章节规则

Each concept has **one canonical chapter** — the chapter that owns the definition. Other chapters elaborate, contrast, or apply, but the canonical chapter wins on definitional disputes.
每个概念有**唯一的规范章节**——拥有该概念定义权的章节。其他章节负责扩展、对照或应用，但定义之争以规范章节为准。

**Rules:**
- A page's `canonical_chapter` field equals the chapter that introduces the concept's full definition, **not** every chapter that mentions it.
- If a concept is genuinely co-introduced (e.g. *cast design* in Ch.8 and Ch.17), the **first** chapter wins canonical authority; the second is `chapter_refs` only.
- Concepts created from supplementary sources (e.g. `law-of-diminishing-returns`) take their canonical chapter from the McKee chapter that anchors the source's argument, with `(supplementary)` noted.
- The Mermaid diagram on each concept page should reflect relationships present in the **canonical** chapter; later-chapter elaborations live in prose.

**规则：**
- 页面的 `canonical_chapter` 字段等于该概念**完整定义**所在章节，而非所有提及的章节。
- 如果概念真的是双章共同引入（例如人物阵容设计在第8章和第17章），**先来者**拥有规范权威，后者仅出现在 `chapter_refs`。
- 来自补充资料的概念（如 `law-of-diminishing-returns`），规范章节取该论证最贴合的麦基章节，并备注 `(supplementary)`。
- 概念页 Mermaid 图反映**规范章节**中的关系；后续章节的扩展应写入文字部分，而非图。

## 3. Conflict Resolution / 冲突解决

When two pages or sections appear to contradict, apply these rules in order:
当两个页面或段落出现矛盾，按以下顺序处理：

1. **Newer chapter elaborates earlier; never overwrites.** If Ch.14 reframes the *gap* in light of antagonism, Ch.7's definition is still canonical; Ch.14's framing is a corollary noted in `chapter_refs` and prose.
   **后续章节扩展，而非推翻早期章节。** 第14章用对抗重新定位"鸿沟"，第7章的定义仍为规范，第14章的视角作为推论列入 `chapter_refs` 与正文。

2. **McKee's word over wiki synthesis.** If a Notable Quote contradicts a wiki paraphrase, the quote wins; rewrite the paraphrase. Quotes are now atomic — see `wiki/{en,zh}/quotes/`.
   **麦基原话高于维基综述。** 如果"重要引文"与维基复述矛盾，以原话为准，重写复述。引文已被原子化——见 `wiki/{en,zh}/quotes/`。

3. **Bilingual mismatches: ZH must align to EN content, but adapt to ZH reading conventions.** A topology mismatch in Mermaid diagrams between EN/ZH counts as a bug, not a translation choice.
   **双语错位：中文版必须与英文版内容对齐，但可顺应中文阅读习惯。** EN/ZH Mermaid 图拓扑结构不一致属于 bug，不是翻译选择。

4. **Importance scoring breaks ties.** When two concepts compete for which is "primary" in a relationship diagram, the one with higher `importance` is the parent node.
   **重要度分级用于打破平局。** 当两个概念竞争图中"主要节点"时，`importance` 较高者为父节点。

5. **When in doubt: flag, don't overwrite.** Add a `<!-- CONFLICT: ... -->` HTML comment in the page body and add a `contradictions flagged` line in the next log entry. Resolution is the user's call.
   **拿不准时：标记，不要覆盖。** 在页面正文中加 `<!-- CONFLICT: ... -->` HTML 注释，并在下一次日志条目中加 `标记矛盾` 行。最终决议由用户作出。

## 4. Authorship Boundary / 作者权属

Every page has an `author` field of either `claude` or `user`.
每个页面 `author` 字段为 `claude` 或 `user` 之一。

| `author: claude` | `author: user` |
|---|---|
| All pages outside `notes/` — chapters, concepts, structures, principles, comparisons, applications, characters, entities, genres, quotes. Claude owns these; the user can edit freely, but Claude is responsible for keeping them current. | All pages inside `wiki/{en,zh}/notes/`. These are personal reflections from the human writer. **Claude must not silently rewrite their content.** |
| Update freely on every operation. | Update only when the user explicitly requests, or to fix typos / dead wikilinks. Substantive content changes require the user's confirmation. |
| `notes/` 之外的所有页面——章节、概念、结构、原则、对比、应用、角色、实体、类型、引文。Claude 拥有这些页面；用户可自由编辑，但 Claude 负责保持其时效。 | `wiki/{en,zh}/notes/` 中的所有页面。这些是人类作者的个人反思。**Claude 不得擅自重写其内容。** |
| 每次操作可自由更新。 | 仅在用户明确请求时更新，或修正笔误/失效的 wikilink。实质性内容修改需用户确认。 |

The `notes/` boundary mirrors what the article calls *identity files* — the user's "soul." Mixing user reflections with Claude-authored synthesis silently is forbidden, because the value of `notes/` is precisely that it is *not* synthesis.
`notes/` 边界对应文章所说的*身份档案*——用户的"灵魂"。把用户反思与 Claude 综述悄悄混合是被禁止的，因为 `notes/` 的价值恰恰在于它*不是*综述。

## 5. Field Glossary / 字段说明

| Field | Type | Purpose |
|---|---|---|
| `importance` | int 1–5 | Retrieval ranking. 5 = foundational concept McKee returns to repeatedly; 1 = minor reference. Defaults by type, with overrides for foundational slugs. |
| `canonical_chapter` | int or null | The single chapter that owns this concept's definition. Used for conflict resolution. |
| `last_verified` | date | Date the page was last cross-checked against its source. Distinct from `last_updated` (which marks any edit). |
| `author` | `claude` or `user` | Authorship boundary (see §4). |

| 字段 | 类型 | 作用 |
|---|---|---|
| `importance` | 1-5 整数 | 检索时的排序权重。5 = 麦基反复回归的基础概念；1 = 次要引用。按类型设默认值，对基础概念按 slug 覆盖。 |
| `canonical_chapter` | 整数或空 | 拥有此概念定义权的唯一章节。用于冲突解决。 |
| `last_verified` | 日期 | 此页面最后一次对照来源核实的日期。与 `last_updated`（任何编辑都会更新）不同。 |
| `author` | `claude` 或 `user` | 作者权属边界（见 §4）。 |

## 6. Operations Touching Canonical / 涉及规范的操作

- **INGEST** must set `canonical_chapter`, `importance`, `last_verified` on every new page.
- **LINT** must verify (a) every page has an `author`; (b) `last_verified` is not older than the source file's mtime; (c) terminology in the page matches §1; (d) `canonical_chapter` is consistent with `chapter_refs`.
- **QUERY** should weight `importance` when ranking results.

- **收录** 操作必须为每个新页面设置 `canonical_chapter`、`importance`、`last_verified`。
- **检查** 操作必须验证：(a) 每个页面有 `author` 字段；(b) `last_verified` 不旧于来源文件的修改时间；(c) 页面中的术语与 §1 一致；(d) `canonical_chapter` 与 `chapter_refs` 不相悖。
- **查询** 操作应将 `importance` 纳入结果排序。
